A walkthough of the collaberative filtering method - The main ideas and the mathematics

Technically, the type of recommender system we will implement is called collaborative filtering. We will construct at Matrix with the shape \( (# users)x(# games) \). This matrix contains all the votes and game ownerships for each user. The matrix is incomplete in the sense that not every user will have voted for every game. The aim is to establish which games a user might like, that is, we will fill out the missing parts of the \(R\) matrix. $$ \theta\cdot X = R^* $$ We do this by finding the matrixes \( X \) and \( \Theta \) whose inner product gives the filled out matrix \( R^* \).

To ge the matrices \( X \) and \( \Theta \) we could simply try out numbers untill we hit ones with a reasonable inner product. But this would be a tedious task. What we want is for the computer to do all the heavy lifting. For this we must come up with an expression that we can then minimize. This is expression is usually called the cost function, presumably because it is the penalty the coomputer must suffer for putting up a wrong answer. The cost function we will use is: $$ J(X,\Theta, R) = \frac{1}{2} \Sigma \Sigma \left( \Theta\cdot X - Y \right)^{2} + \frac{\lambda}{2} \cdot \Sigma \Theta^{2} + \frac{\lambda}{2} \cdot \Sigma X^{2} $$ The first term is the squared difference between the prediction votes and the acutal votes. The last two term are regularization terms. These are included to help the gradient decent algorithm converge.

Since we are dealing with \( # user + # games \) parameters to be fitted it is nescessary to supply the gradient of the cost function.
